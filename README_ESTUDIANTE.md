# Actividad de aprendizaje 10: Caza errores

## Proyecto: Lúmina Rooftop Bar

Vas a trabajar con un sitio web ficticio de un bar llamado **Lúmina Rooftop Bar**. El sitio tiene diseño visual, carta de bebidas, sección de experiencia y formulario de reservas. Sin embargo, el proyecto contiene errores intencionales en HTML, CSS y JavaScript.

Tu misión es abrir el proyecto, probarlo, encontrar los errores, corregirlos y documentar cada solución.

## Objetivo de la actividad

Identificar, diagnosticar y corregir errores frecuentes en HTML, CSS y JavaScript, aplicando pruebas en navegador, revisión de consola y documentación técnica del proceso.

## Archivos que debes revisar

```text
proyecto_con_errores/
├── index.html
├── css/styles.css
├── js/app.js
└── assets/
```

## Paso a paso de trabajo

### 1. Abrir el proyecto en Visual Studio Code

1. Descomprime el ZIP entregado por el docente.
2. Abre Visual Studio Code.
3. Selecciona **File > Open Folder** o **Archivo > Abrir carpeta**.
4. Abre la carpeta de la actividad.
5. Ubica la carpeta `proyecto_con_errores`.

### 2. Ejecutar el proyecto en Google Chrome

1. Abre el archivo `index.html`.
2. Haz clic derecho y selecciona **Open with Live Server**, si tienes la extensión instalada.
3. Si no tienes Live Server, abre directamente el archivo `index.html` con Google Chrome.

### 3. Observar los errores

Revisa si ocurre alguno de estos problemas:

- La imagen principal no carga.
- El menú no lleva a la sección correcta.
- El diseño de las tarjetas no se ve profesional.
- Los filtros de la carta no funcionan.
- El formulario no muestra confirmación.
- La consola del navegador muestra errores.
- El diseño responsive no funciona correctamente en pantalla pequeña.

### 4. Revisar la consola

En Google Chrome:

1. Presiona `F12` o clic derecho > **Inspeccionar**.
2. Abre la pestaña **Console** o **Consola**.
3. Anota los errores que aparezcan.
4. Relaciona cada error con el archivo que lo produce.

### 5. Corregir el código

Corrige los errores en:

- `index.html`
- `css/styles.css`
- `js/app.js`

No borres secciones completas sin justificar. La idea es diagnosticar y corregir, no reconstruir todo desde cero.

### 6. Documentar el proceso

Completa la plantilla `PLANTILLA_BITACORA_CORRECCION.md` con:

- Error encontrado.
- Archivo afectado.
- Diagnóstico.
- Corrección aplicada.
- Evidencia.
- Resultado final.

### 7. Resolver el cuestionario

Responde el archivo `CUESTIONARIO_ESTUDIANTE.md`.

### 8. Crear mapa mental, mapa conceptual o notas técnicas

Usa la guía `GUIA_MAPA_MENTAL_NOTAS.md` para organizar los errores encontrados y las soluciones.

## Entregables

Debes entregar en Moodle:

1. Carpeta del proyecto corregido en formato `.zip`.
2. Bitácora de corrección diligenciada.
3. Cuestionario resuelto.
4. Mapa mental, mapa conceptual o notas técnicas.
5. Evidencias de prueba:
   - Captura del proyecto antes de corregir.
   - Captura del proyecto corregido.
   - Captura de la consola antes o durante el diagnóstico.
   - Captura de la consola final sin errores críticos.
   - Captura del código corregido en VS Code.

## Recomendación de trabajo

Trabaja con orden. Corrige un error a la vez, guarda los cambios y vuelve a probar en el navegador. Si cambias muchas cosas al mismo tiempo, será más difícil saber qué funcionó y qué generó un nuevo problema.
