# Rúbrica de evaluación

| Criterio | Superior | Alto | Básico | Bajo |
|---|---|---|---|---|
| Diagnóstico de errores | Identifica errores HTML, CSS y JS con explicación clara y técnica. | Identifica la mayoría de errores y explica sus causas. | Identifica algunos errores, pero con explicación incompleta. | No identifica errores relevantes o confunde las causas. |
| Corrección del código | Corrige los errores sin generar nuevos problemas y mantiene el proyecto ordenado. | Corrige la mayoría de errores y el sitio funciona adecuadamente. | Corrige algunos errores, pero quedan fallos visibles o funcionales. | El proyecto no funciona o las correcciones son insuficientes. |
| Pruebas y evidencias | Presenta evidencias completas: antes, después, consola y código. | Presenta la mayoría de evidencias solicitadas. | Presenta evidencias incompletas. | No presenta evidencias claras. |
| Bitácora técnica | Documenta error, causa, solución, prueba y resultado con claridad. | Documenta los elementos principales. | Documenta de forma superficial. | No documenta o la información no es comprensible. |
| Cuestionario | Responde con análisis propio y vocabulario técnico. | Responde adecuadamente la mayoría de preguntas. | Responde de forma breve o incompleta. | No responde o copia sin evidenciar comprensión. |
| Mapa mental/notas | Organiza visualmente errores, diagnóstico y soluciones. | Presenta organización clara con algunos detalles técnicos. | Presenta ideas básicas poco conectadas. | No presenta mapa/notas o no se relaciona con la actividad. |

## Ponderación sugerida

| Componente | Porcentaje |
|---|---:|
| Proyecto corregido | 35% |
| Bitácora técnica | 20% |
| Evidencias de prueba | 15% |
| Cuestionario | 15% |
| Mapa mental, conceptual o notas | 10% |
| Orden de entrega | 5% |
