# Cuestionario de la actividad

Responde con tus propias palabras. Puedes apoyarte en el proyecto, la consola del navegador y tu bitácora de corrección.

## Parte 1: Reconocimiento de errores

1. ¿Qué errores visuales identificaste al abrir por primera vez el sitio web?
2. ¿Qué errores aparecieron en la consola del navegador?
3. ¿Qué diferencia hay entre un error de HTML, un error de CSS y un error de JavaScript?
4. ¿Por qué una imagen puede no cargar aunque el archivo exista dentro del proyecto?
5. ¿Qué puede pasar si el nombre de una clase en HTML no coincide con el selector escrito en CSS?

## Parte 2: Diagnóstico técnico

6. ¿Qué herramienta del navegador usaste para revisar los errores?
7. ¿Cómo verificaste si una ruta de imagen, CSS o JavaScript estaba bien escrita?
8. ¿Cómo comprobaste si los estilos CSS se estaban aplicando correctamente?
9. ¿Qué significa que JavaScript devuelva un error relacionado con `null`?
10. ¿Por qué es importante revisar los identificadores `id`, las clases y los atributos `data-*`?

## Parte 3: Corrección y pruebas

11. Describe tres errores que corregiste y explica cómo los solucionaste.
12. ¿Qué pruebas realizaste después de corregir el código?
13. ¿Cómo verificaste que el formulario de reservas funcionaba correctamente?
14. ¿Qué comprobaste en la versión responsive o en pantalla pequeña?
15. ¿Qué evidencia demuestra que el sitio quedó funcionando mejor que al inicio?

## Parte 4: Reflexión

16. ¿Cuál fue el error más difícil de encontrar? ¿Por qué?
17. ¿Qué aprendiste sobre la relación entre HTML, CSS y JavaScript?
18. ¿Por qué es importante documentar los errores y soluciones?
19. ¿Qué buenas prácticas aplicarías en tu próximo proyecto web?
20. ¿Cómo puede ayudarte esta actividad cuando desarrolles un proyecto real?
